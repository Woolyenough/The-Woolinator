# Privacy Policy — The Woolinator

_Last updated: 25 June 2026_

The Woolinator ("the bot", "we", "us") is a self-hosted, open-source Discord bot operated by [**`Woolyenough`**](https://github.com/Woolyenough) (Discord: `@woolyenough`). This policy explains what data the bot processes, why, how long it is kept, and how you can have it removed. By using the bot you agree to this policy.

This policy adheres to Discord's
[Developer Terms of Service](https://discord.com/developers/docs/policies-and-agreements/developer-terms-of-service)
and [Developer Policy](https://discord.com/developers/docs/policies-and-agreements/developer-policy).

## Who operates the bot

The bot is run on hardware controlled by the operator (Woolyenough). Data is not sold, rented, or shared for advertising. Access to the bot's database and host is restricted to the operator.

## Data we store

We only collect information when it is needed to provide a feature you choose to use, and only by fair means and with your knowledge.

The following may be saved to the bot's database, solely so its features work properly:

- **User IDs**: to link you and your server members with your saved configurations
- **Server IDs**: for server-specific configuration and features
- **Channel IDs**: for configurations that are specific to channels
- **Role IDs**: for configurations that are specific to roles
- **Message IDs**: for message-specific features
- **Emoji IDs**: for features that react to or recognise specific emoji
- **Any user-supplied content**: text you provide, which can include reminders, birthdays, and tags

We do **not** store your messages. We never ask for credentials, and you should never give them to any bot.

## Data we process but do **not** store

To carry out certain features, the bot reads some information only at the moment it is needed and does not save it. This can include message content, member details, and presence (such as online status or activity). Such information is shown in Discord or held only in memory to perform the requested action, and is discarded afterwards or when the bot restarts.

The main example is when the bot forwards information within a server rather than storing it; for instance, logging functionality, where message activity is relayed to a server's own log channel without being saved to the bot's database.

## Third parties

- **Discord**: the platform the bot runs on, governed by Discord's own
  [Privacy Policy](https://discord.com/privacy).
- **External APIs**:
  - `/define` → Urban Dictionary API (your search term)
  - `/insult` → insult.mattbas.org (the target member's **username**)

## How to delete your data

You can view and remove most of your data yourself, at any time, with the command: `/data-review`
